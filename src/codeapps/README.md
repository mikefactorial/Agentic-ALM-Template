# src/codeapps

Power Apps Code Apps — standalone React+Vite web applications hosted in Power Platform with access to Power Platform connectors and Dataverse data. One subfolder per solution area.

## What Lives Here

```
src/codeapps/
  acm_AcmePlatform/
    ContractReview/
      src/
        App.tsx           # Root React component
        main.tsx          # Entry point
      dist/               # Compiled static assets (generated by npm run build)
      vite.config.ts      # Must disable content hashing for stable map.xml paths
      package.json
      tsconfig.json
      power.config.json   # Created by pac code init — links to Dataverse app record
```

## What Code Apps Are

Code Apps are full single-page applications (React + Vite) that run hosted inside Power Platform. Unlike PCF controls (embedded in a form field) or web resources (scripts called by the platform), code apps are standalone apps navigated to directly — similar to a model-driven or canvas app, but built with standard web tooling.

Key characteristics:
- React + TypeScript development experience
- Access to Power Platform connectors (Dataverse, SharePoint, custom APIs, etc.)
- Packaged inside a Dataverse solution alongside other components
- Require admin enablement per environment and a Power Apps Premium license for end users

## When to Use

| Need | Use |
|------|-----|
| Standalone app with connectors and rich UI | Code app (this folder) |
| Custom UI component on a model-driven form | PCF control (`src/controls/`) |
| Form/ribbon script logic | Web resource (`src/webresources/`) |

## Prerequisites

- **Admin must enable code apps** on each target environment: Settings > Product > Features > Power Apps code apps
- **End users need a Power Apps Premium license** to run code apps
